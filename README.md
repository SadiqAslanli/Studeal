The backend for this platform is built entirely within the Next.js framework and utilizes Supabase as the primary database for real-time data management and authentication. The architecture relies on a clear hierarchy starting with the Admin who holds full system authority to create and manage both Company and Student user accounts. Companies serve as the content layer of the platform with the ability to publish topics and posts that engage the community while also distributing specific coupons and discounts. The Students represent the end-user group who interact with the website to discover and redeem the deals provided by the Companies.

By using the Supabase client within Next.js Server Components and Route Handlers, the system can efficiently manage these role-based permissions and data relationships. Each action is verified against the user's role stored in the Supabase Auth metadata to ensure that only authenticated Companies can post content and only verified Students can access the discount logic. This setup allows for a highly scalable environment where the Admin maintains oversight of all accounts while the data flows seamlessly between the corporate providers and the student consumers.
